# 互联网金融信贷系统 — 功能架构设计

> 本文档由架构视角梳理互联网金融信贷系统应具备的功能模块，并**高亮核心功能**，作为系统搭建第一步的蓝图。

---

## 一、功能全景概览

互联网金融信贷系统围绕「获客 → 评估 → 授信 → 放款 → 贷后 → 催收 → 清收」全生命周期，需覆盖用户侧、业务侧、风控侧、资金侧与合规侧。以下按域划分并标注**核心功能**。

---

## 二、按域划分功能清单

### 1. 用户与客户管理

| 功能 | 说明 | 核心 |
|------|------|:----:|
| 用户注册/登录 | 手机号、证件、密码、验证码、生物识别等 | ✓ |
| 实名认证 | 姓名、身份证、银行卡四要素/三要素校验 | ✓ |
| 客户信息管理 | 基本信息、职业、收入、负债、联系人等档案 | ✓ |
| 客户分级/标签 | 价值分层、风险标签、行为标签，支撑差异化策略 | |
| 账户与授权 | 多账户、子账户、授权委托（如代扣授权） | ✓ |

## 三、核心功能汇总（高亮）

以下为**必须优先建设**的核心能力，是信贷业务闭环与风控合规的基础：

1. **用户与实名**：注册/登录、实名认证、客户档案、账户与授权  
2. **产品与定价**：产品配置、定价/费率引擎、额度策略  
3. **进件与风控**：进件申请、反欺诈、信用评估/评分、授信审批、额度生效  
4. **支用与放款**：支用申请、放款审核、电子合同、资金路由与放款执行、结果通知  
5. **贷中与还款**：借据与还款计划、主动还款、代扣、逾期与借据状态  
6. **催收与清收**：任务分配、催收执行、减免/重组、核销与坏账  
7. **支付与资金**：支付渠道、放款/还款通道、对账  
8. **风控与合规**：规则引擎、征信与数据源、名单与限额、合规审计与客户权益  
9. **运营与数据**：审批工作台、报表看板、业务参数配置  
10. **系统基础**：身份与权限、高可用与安全、监控与运维  

---

## 四、建议实施顺序（第一步之后）

- **Phase 1**：用户与实名 + 产品与定价 + 进件与授信 + 规则引擎/评分  
- **Phase 2**：支用与放款 + 借据与还款 + 支付与对账  
- **Phase 3**：催收与清收 + 委外/法催 + 核销  
- **Phase 4**：运营报表、合规报送、多租户与弹性
